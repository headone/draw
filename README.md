# 蹬图浪子：HeadOne 作图页征集活动

用 `api.headone.fit` 做一个作图页面，让它好用、好看、好玩。

作品一经选中，将正式挂载到 `draw.headone.fit`，并获得 **$1000 HeadOne 额度**。只要提交有效作品，即可获得 **$100 HeadOne 额度**。

## 活动要求

- 页面必须基于 `api.headone.fit` 完成作图能力。
- 代码需以 Pull Request 形式提交到本仓库。
- 不限制技术栈，不限制设计风格。
- 可以是工具、实验页面、创意玩法、模板生成器、海报生成器等。
- 页面需可正常运行，体验完整，交互清晰。
- 不得接入非 HeadOne 的作图 API 作为核心生成能力。

## 评选参考

- 创意是否有记忆点。
- 作图体验是否顺滑。
- 页面是否美观、稳定、易用。
- 是否充分发挥 HeadOne API 能力。
- 是否适合挂在 `draw.headone.fit` 长期展示。

## 奖励

- **入选作品**：$1000 HeadOne 额度，并展示在 `draw.headone.fit`。
- **有效参与**：$100 HeadOne 额度。

## 提交方式

请向本仓库提交 Pull Request，并在 PR 描述中包含：

- 作品名称
- 作品简介
- 页面入口 / 运行方式
- 作者名称 / 联系方式
- 使用 `api.headone.fit` 的方式说明

## 参与流程

1. Fork 本仓库。
2. 在自己的分支中完成作品。
3. 确认作品可本地运行，并补充必要说明。
4. 提交 Pull Request。
5. 等待评审与反馈。

## PR 建议

- 保持变更聚焦，一个 PR 对应一个作品。
- 在 README 或 PR 描述中写清楚启动方式。
- 如需环境变量，请提供 `.env.example` 或清晰说明。
- 如有截图或演示链接，欢迎放在 PR 描述里。

## 版权与展示

作者保留作品署名权。提交 PR 即表示同意 HeadOne 在活动评审、结果展示和 `draw.headone.fit` 挂载中使用、展示和传播该作品。
